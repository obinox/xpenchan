# 엑펜쨩봇 (Xpen chan Bot)

TAK의 VOCALOID 곡에서 이름을 따온, 이모티콘 반응으로 역할을 부여하고 관리하는 간단한 디스코드 봇입니다.

## 주요 기능

-   **반응 역할:** 관리자가 설정한 특정 메시지에 사용자가 이모티콘으로 반응하면 해당 역할을 자동으로 부여/회수합니다.
-   **/ 슬래시 명령어:** 디스코드 표준을 준수하는 슬래시 명령어로 봇을 제어합니다.
-   **메시지 내 데이터 저장:** 별도의 데이터베이스나 JSON 파일 없이, 봇이 보낸 메시지 자체에 역할 정보를 저장하여 상태를 유지합니다.
-   **동적 역할 관리:** 역할을 추가하거나 제거하는 명령어로 기존 역할 부여 메시지를 쉽게 수정할 수 있습니다.
-   **자동 정리:** 허용된 이모티콘 외의 반응이 달리면 봇이 자동으로 삭제하여 메시지를 깔끔하게 유지합니다.

## 1. 봇 설정 방법

### 가. 사전 준비

-   [Node.js](https://nodejs.org/)가 설치되어 있어야 합니다.
-   봇을 적용할 디스코드 서버가 필요합니다.

### 나. 프로젝트 설치

1.  이 프로젝트를 컴퓨터에 다운로드합니다.
2.  프로젝트 폴더에서 터미널을 열고, 아래 명령어를 실행하여 필요한 라이브러리를 설치합니다.
    ```bash
    npm install
    ```

### 다. `.env` 파일 설정

프로젝트의 루트 디렉토리(최상위 폴더)에 `.env` 파일을 생성하고 아래와 같은 형식으로 내용을 채워야 합니다.

```env
DISCORD_TOKEN=YourDiscordBotTokenHere
GUILD_ID=YourGuildIdHere
CLIENT_ID=YourClientIdHere
```

-   **`DISCORD_TOKEN`**: [Discord 개발자 포털](https://discord.com/developers/applications)에서 생성한 봇의 토큰입니다.
    -   `New Application` -> `Bot` 탭 -> `Reset Token`
-   **`CLIENT_ID`**: 봇의 클라이언트 ID입니다.
    -   `General Information` 탭에서 `Application ID`를 복사하세요.
-   **`GUILD_ID`**: 봇을 적용할 서버(길드)의 ID입니다.
    -   디스코드 설정에서 `고급 > 개발자 모드`를 활성화하세요.
    -   서버 아이콘을 마우스 오른쪽 버튼으로 클릭하고 `서버 ID 복사`를 선택하세요.

### 라. 봇 초대하기

1.  **권한 설정**: 봇은 역할 관리, 메시지 관리 등 다양한 권한이 필요합니다. 아래 링크는 "엑펜쨩봇"에 필요한 최소한의 권한(`268774464`)을 계산한 초대 링크입니다.

2.  **초대 링크 접속**: 아래 URL에서 `YOUR_CLIENT_ID` 부분을 위에서 발급받은 봇의 `CLIENT_ID`로 교체한 후, 웹 브라우저로 접속하여 봇을 서버에 초대하세요.

    ```
    https://discord.com/api/oauth2/authorize?client_id=YOUR_CLIENT_ID&permissions=268774464&scope=bot%20applications.commands
    ```
    
3.  **Privileged Intents 활성화**: Discord 개발자 포털의 'Bot' 탭에서 **MESSAGE CONTENT INTENT**를 **반드시** 활성화해주세요.

## 2. 봇 실행 및 사용법

### 가. 명령어 등록

봇의 슬래시 명령어를 실제 서버에 등록하는 과정입니다. 이 작업은 `commands` 폴더의 내용을 수정했을 때 한 번만 실행해주면 됩니다.

```bash
npm run deploy
```

### 나. 봇 실행

아래 명령어로 봇을 실행합니다.

```bash
npm start
```

봇이 성공적으로 로그인하면 터미널에 "Ready! Logged in as [봇이름]" 메시지가 출력됩니다.

### 다. 명령어 사용법

#### `/setup`
역할 부여 메시지를 처음 생성합니다.

-   **`roles`**: `이모티콘 @역할` 형식으로 짝을 지어 입력합니다. 여러 개를 동시에 설정할 수 있습니다. (필수)
-   **`channel`**: 메시지를 보낼 채널을 지정합니다. (선택, 기본값: 현재 채널)
-   **`title`**: 임베드 메시지의 제목을 정합니다. (선택, 기본값: 역할 선택)

**예시:**
`/setup roles: ✨ @디자이너 ⭐ @개발자`

#### `/add-role`
기존에 생성된 역할 부여 메시지에 새로운 역할-이모티콘 쌍을 추가합니다.

-   **`message-id`**: 수정할 역할 부여 메시지의 ID. (메시지 우클릭 > `메시지 ID 복사`로 확인) (필수)
-   **`emoji`**: 추가할 이모티콘. (필수)
-   **`role`**: 추가할 역할. (필수)

**예시:**
`/add-role message-id: 123456789012345678 emoji: 🎵 role: @프로듀서`

#### `/remove-role`
기존 역할 부여 메시지에서 특정 역할-이모티콘 쌍을 제거합니다.

-   **`message-id`**: 수정할 역할 부여 메시지의 ID. (필수)
-   **`emoji`**: 제거할 이모티콘. (필수)

**예시:**
`/remove-role message-id: 123456789012345678 emoji: ✨`

#### `/create-role`
권한이 없는 빈 역할을 새로 생성합니다. 역할 부여 기능과 별개로 사용할 수 있습니다.

-   **`name`**: 생성할 역할의 이름. (필수)
-   **`color`**: 역할의 색상. `#FF0000` 같은 16진수 코드를 사용하거나 `Red`, `Blue` 등의 기본 색상 이름을 사용할 수 있습니다. (선택)

**예시:**
`/create-role name: 방문객 color: #808080`
